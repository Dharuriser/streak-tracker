# 肌ルーティン（Skin Care Checker）

朝・夜に使うものと順番を、時間帯と肌状態に合わせて案内するサーバー不要の静的Webアプリです。記録は評価や罰ではなく、実際に行ったケアを覚えておくための補助として扱います。

記録は利用中のブラウザの `localStorage` だけに保存され、外部には送信されません。

## ローカルで確認する

Service Workerは `file://` では動作しないため、簡易HTTPサーバー経由で開いてください。

```sh
python3 -m http.server 8080
```

ブラウザで <http://localhost:8080> を開きます。

## 仕様

- 朝は04:00〜15:59、夜は16:00〜翌03:59です。深夜0:00〜3:59の夜ケアは前日分として記録します。
- 朝は毎日、洗顔・MISSHA・Ceramedx・Badgerの日焼け止めを表示します。
- 夜は毎日、洗顔から始まります。
- 大きな「やった」ボタンを押すだけで、その時間帯の手順をまとめて完了にできます。
- 記録日と朝／夜を切り替えると、押し忘れた記録をあとから「やった」「やってない」に補正できます。
- 肌状態と500文字までの任意メモを、朝／夜ごとに残せます。
- 合計実行回数・今週の実行数・今月の実行数を主な指標として表示します。
- 連続日数は、最後に成立した連続記録を参考値として表示します。直近の押し忘れだけで0に戻しません。
- 初回利用日から30日間を1か月目、31日目以降を2か月目として扱います。
- 2か月目の火曜は、直近7日間に肌トラブルがなければThayersの代わりにAdvancedを週1回表示します。
- 日曜のAHAは、その時間帯の肌状態で「問題なし」を選んだ場合だけ表示します。
- 乾燥・赤み・ヒリつき・皮むけの記録後3日間は、夜の刺激成分を表示しません。
- 日焼けを記録した当日と翌日は、夜の刺激成分を表示しません。
- Advanced使用後2日以内に赤み・ヒリつき・皮むけが記録された場合、使用日から14日間Advancedを停止します。
- 朝／夜の達成率は補助情報として小さく表示し、最大直近30日を対象に計算します。

ルーティン判定の詳細は [`docs/routine-rules.md`](docs/routine-rules.md)、アイテム一覧は [`docs/skincare-items.md`](docs/skincare-items.md) にまとめています。

## バックアップ

右上の「•••」から、JSON形式で記録を書き出し・読み込みできます。読み込みは現在の記録を置き換えます。ブラウザのサイトデータを消すと記録も消えるため、定期的な書き出しを推奨します。

## GitHub Pages

すべて相対パスで構成しているため、リポジトリ直下の内容をそのままGitHub Pagesへ公開できます。アクセストークンやGitHub APIは使用していません。

1. GitHubのリポジトリ設定で **Pages** を開く
2. 公開元のブランチと `/ (root)` を選択
3. 発行されたHTTPSのURLへアクセス

PrivateリポジトリでのPages利用可否はGitHubプランに依存します。公開前にリポジトリとPagesの公開範囲を確認してください。

## ファイル

- `index.html` — 画面構造
- `style.css` — スマホ優先のUI
- `script.js` — ルーティン判定、記録、統計、バックアップ
- `manifest.json` — ホーム画面追加用のPWA設定
- `service-worker.js` — アプリシェルのオフラインキャッシュ
- `icon.svg` / `icon-192.png` / `icon-512.png` — アプリアイコン
- `docs/routine-rules.md` — 曜日・安全ルール
- `docs/skincare-items.md` — 使用アイテムと役割
